## 2. Основание для разработки
Необходимость переноса исторических данных о штрихкодах из УТ 10, где они были привязаны к конкретным организациям и контрагентам, в УТ 11.5 с сохранением функциональности и возможностью использования этих данных в печатных формах без модификации типовых механизмов работы со штрихкодами.

## 3. Цель и назначение

*   **Цель:** Создать механизм хранения и использования расширенных данных о штрихкодах (с привязкой к организации и контрагенту) в УТ 11.5, совместимый с существующей системой штрихкодирования.
*   **Назначение:** Функционал предназначен для службы логистики, склада и отдела продаж для работы с историческими данными и специализированными сценариями маркировки.



## 4. Функциональные требования
* Создать обработку "Перенос штрихкодов из УТ10"
* Создать расширение "Управление расширенными штрихкодами" для ручного ввода, редактирования и удаления записей в регистре.
* Обеспечить возможность использования данных из регистра в печатных формах
