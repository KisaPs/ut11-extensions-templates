Работая на стороне ИФЗ, теряется цепока связей заказов приобретений и товаров, документ не будет создан в УТ

Необходимо рабочее место для компоновки товара перед созданием заказа поставщику. Вывести предварительный список создаваемых документов.
Решение: Выделить пространство(промежуточную форму) для графического отображения, сортировки товара на несколько заказов поставщику.
Удобное разделение товарного состава на группы

Алгоритм.
В заказ клиента добавляем кнопку с командой Распределить заказ. По нажатию, показывается  форма распределения.
В форму загружается заказ клиента, так же настройки распределения складов по организации из Регистра Настройки распределения заказа. Заказ клиента должен оставаться в резерве(как есть) на момент проведения операций в форме.
После загрузки ЗК и настроек, необходимо обновить склады (колонка отправитель) в соответствии с заданными настройками. Как реализовано в заказе клиента по кнопке. Так же иметь возможность изменить склады и обновить в сворачиваемом окне. В ТЧ Формы все колонки выводить пользователю необязательно, оставив только идентификацию номенклатуры и её МГХ.
В окно 1, выводим строки сгруппированые по полю Склад и СкладИФЗ. АГрегируем Количество, объем.
Необходимые две команды - Распределить заказ, Распределить Все заказы и Группа команд автоматического распределения, на данные момент одна команда - распределить по объему.
Окно 2, содержит распределенные грузоместа (будущие заказы поставщику), и при необходимости их можно добавить/удалить.
Окно 3, таблица нераспределенных товаров выделенной строки из Окна 1.
Окно 4, таблица распределенных товаров выделенной строки из Окна 1. 
Порядок действий для менеджера
Менеджер в заказе , по нажатию кнопки, открывает форму распределения. Срабатывает проверка, на заполненные скалады ИФЗ и отсутствие созданных, проведенных заказов поставщику.

Форма автоматически заменяет склады по настройкам, при желании менеджер может изменить настройку. Которая очищает заполненное распределение, если таковое имеется.
В окне1 М видит обязательную структуру, минимальную к распределению - группировка Склад+СкладИФЗ.
Нажав на строку ТЧ окно1, командами создать грузоместо в Окне2 ставит столько строк, сколько необходимо создать Заказов поставщику на одну строку из окна1.
В окнах 3 и 4, М командами заполняет грузоместа товарами. Все товары из строки Окна1, по строкам создаваемых товаров Окна2.
По завершению распределения, нажимает Кнопку завершить.
Команды
Завершить - Выполняется проверка на распределенные товары. Если какое то количество не распределено - Вывести предупреждение.
Далее в транзакции необходимо произвести ряд действий. В заказе клиента, необходимо изменить склады в соответствии с настройками (уже из формы распределения, тк М мог их изменить). Провести Заказ Клиента.
Перед созданием заказов поставщику, выполнить проверку на доступное количество к заказу с учетом уже зарегистрированных товаров в Регистре накопления Резервы ИФЗ. С учетом резерва текущего Заказа клиента! В случае отстутствия нужного количества - вывести предупреждение, транзакцию откатить.
На основании текущего заказа клиента, создать Столько заказов Поставщику, сколько было в распределении с учетом распределенных товаров и обособления их.
При успешном завершении проведения Заказ поставщку, вывести список ссылок на созданные заказы.

Распределить по объему - Автоматическая операция, распределения товаров по количеству  грузомест и товаров в них.
При нажатии, выполняется проверка на заполненный реквизит формы  Введите объем (последнее введенное значение, необходимо подставлять автоматом). Проверка на заполненный реквизит объема распределяемой номенклатуры, вывести предупреждение если в номенклатуре реквизит не заполнен, процесс остановить.
По алгоритму, создать грузоместа и заполнить товарами в них.
