# Техническое задание: Модификация работы с одним заказом клиента

## 1. Общие требования
- Разработку вести в **отдельной Git-ветке**.
- Основная цель: адаптация механизмов под работу с одним заказом клиента (без владельцев остатков и лишних договоров).

## 2. Изменение распределения Коммерческого предложения (КП)
Новый принцип распределения:
1. **Разделить организации продавца**.
2. **Отделить товары «Не в наличии»**.

> Товары, которых нет в наличии, также выделяются в отдельное КП для принятия решения ответственному менеджеру.

## 3. Склады в заказе клиента

- **Убираются**: `Владелец остатков`, `Договор владельца`, `Договор продавца`.

- **Склад в ТЧ** привязывается к складу шапки (зависимый выбор).

- признак обособленности
- **Склад ИФЗ**

## 5. Принцип работы механизма «Распределить по складам»

### 5.1. Действие по кнопке
1. Процедура ищет **первую настройку** в справочнике по условию:
   - `Организация из Заказа` == `Организация в настройке`
2. Если настройка найдена:
   - перезаполняет `Склад шапка`
   - перезаполняет в ТЧ товары: `Склад`, `Обособленность`, `Обеспечение`, `Склад ИФЗ` (новое поле)
   - если в настройке `Склад ИФЗ` пуст → ставим пустое поле
3. Если настройка для организации **отсутствует** → выдать сообщение пользователю.

## 6. Создание Заказа поставщику

### 6.1. Изменение логики выгрузки
- Убрать галочку **«Выгружать в ИФЗ»**.
- Добавить кнопку (одно из названий):
  - «Поставить в очередь выгрузки»
  - «Сформировать заявку в ИФЗ»

### 6.2. Группировка товаров (важно!)
Товары в Заказе клиента агрегируются **по полю `Склад ИФЗ`**:
- Пустые значения `Склад ИФЗ` → пропускаем (не создаём документ).
- На каждый получившийся `Склад ИФЗ` → создаётся **отдельный Заказ поставщику**.

### 6.3. Заполнение Заказа поставщику, поля заполняются как и прежде только
| Поле                | Значение                                                                               |
|---------------------|----------------------------------------------------------------------------------------|
| `Склад ИФЗ`         | заполняется из ТЧ Заказа клиента                                                       |
| Комментарий         | `Номер ЗК`, `Дата`, `Комментарий из Заказа клиента`, `Представление склада ИФЗ (первый)` |

### 6.4. Фоновая процедура кнопки
1. Создать Заказы поставщику (типовые реквизиты остаются прежними).
2. Установить признак выгрузки в ИФЗ.

### 6.5. Блокировка повторного создания
После успешного создания Заказов поставщику — кнопка **блокируется** (повторное создание документов невозможно).
