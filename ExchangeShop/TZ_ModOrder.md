# Техническое задание: Модификация работы с одним заказом клиента

## 1. Общие требования
- Разработку вести в **отдельной Git-ветке**.
- Основная цель: адаптация механизмов под работу с уникальным заказом клиента (без «владельцев остатков» и лишних договоров).

## 2. Изменение распределения Коммерческого предложения (КП)
Новый принцип распределения:
1. **Разделить организации продавца**.
2. **Отделить товары «Не в наличии»**.

> Товары, которых нет в наличии, не участвуют в дальнейших отгрузочных сценариях.

## 3. Склады в заказе клиента
### 3.1. Изменение настроек складов
- **Убираются**: `Владелец остатков`, `Договор владельца`, `Договор продавца`.
- **Добавляется**: `Склад шапка` (допускается выбор группы складов).
- **Склад в ТЧ** привязывается к складу шапки (зависимый выбор).

### 3.2. Поведение при распределении КП (внутри сделки)
- Создаются документы, **отличие только в складе в ТЧ товары**.
- Склад шапка и склад ТЧ `Товары` подбираются согласно новым настройкам.

### 3.3. Новое поле в ТЧ
В табличные части **Заказ клиента** и **Заказ поставщику** добавить поле:
| Имя поля     | Тип                      |
|--------------|--------------------------|
| `Склад ИФЗ`  | Справочник.Склады (ссылка) |

---

## 4. Справочник «Настройки заполнения складов в заказе клиента»
Уже реализован, работает. Требуется доработка.

### 4.1. Текущий состав
- `Название`
- `Организация`
- ТЧ: `Склад источник`, `Склад назначения`, `Вариант обеспечения`, `Признак обособленности`

### 4.2. Что добавить
| Новое поле                     | Поведение                                                            |
|--------------------------------|----------------------------------------------------------------------|
| `Склад для шапки Заказа`        | –                                                                    |
| `Склад назначения`              | сделать **зависимым от склада шапки**                                |
| `Склад ИФЗ` (третий склад)      | новое поле, может быть пустым                                        |

### 4.3. Основное назначение справочника
Автоматическое перезаполнение по кнопке **«Распределить по складам»**:
- реальные склады отгрузки в ТЧ (товары)
- вариант обеспечения
- признак обособленности
- **Склад ИФЗ**

---

## 5. Принцип работы механизма «Распределить по складам»

### 5.1. Действие по кнопке
1. Процедура ищет **первую настройку** в справочнике по условию:
   - `Организация из Заказа` == `Организация в настройке`
2. Если настройка найдена:
   - перезаполняет `Склад шапка`
   - перезаполняет в ТЧ товары: `Склад`, `Обособленность`, `Обеспечение`, `Склад ИФЗ` (новое поле)
   - если в настройке `Склад ИФЗ` пуст → ставим пустое поле
3. Если настройка для организации **отсутствует** → выдать сообщение пользователю.

### 5.2. Когда вызывается
- ✅ После создания распределения КП
- ✅ После ручного/автоматического распределения складов в Заказе клиента

---

## 6. Создание Заказа поставщику

### 6.1. Изменение логики выгрузки
- Убрать галочку **«Выгружать в ИФЗ»**.
- Добавить кнопку (одно из названий):
  - «Поставить в очередь выгрузки»
  - «Сформировать заявку в ИФЗ»

### 6.2. Фоновая процедура кнопки
1. Создать Заказы поставщику (типовые реквизиты остаются прежними).
2. Установить признак выгрузки.

### 6.3. Группировка товаров (важно!)
Товары в Заказе клиента агрегируются **по полю `Склад ИФЗ`**:
- Пустые значения `Склад ИФЗ` → пропускаем (не создаём документ).
- На каждый получившийся `Склад ИФЗ` → создаётся **отдельный Заказ поставщику**.

### 6.4. Заполнение Заказа поставщику
| Поле                | Значение                                                                               |
|---------------------|----------------------------------------------------------------------------------------|
| `Склад ИФЗ`         | заполняется из ТЧ Заказа клиента                                                       |
| Комментарий         | `Номер ЗК`, `Дата`, `Комментарий из Заказа клиента`, `Представление склада ИФЗ (первый)` |

### 6.5. Блокировка повторного создания
После успешного создания Заказов поставщику — кнопка **блокируется** (повторное создание документов невозможно).

---

## 7. Регистр накопления «ИФЗ»

| Действие                                                      | Движение |
|---------------------------------------------------------------|----------|
| Проведение **Заказа поставщику** (только для Партнера ИФЗ)     | `+`      |
| Проведение **Приобретения товаров и услуг** по основанию Заказ поставщику | `-`      |

> Партнер ИФЗ — настраивается в параметрах создания Заказа поставщику.
